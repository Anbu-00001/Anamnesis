After a deep, saturation-checked literature review (2026-06-07), the roadmap for Anamnesis (`/home/anbu/26_class/PlayGround/anamnesis`). **Tier 1 SHIPPED 2026-06-07** (uncommitted — user handles all git push).

**Design principle (the corrected thesis):** the calibration win is *mechanical*, not motivational. Lean on the **recorded track record + a learned recalibration map applied deterministically** ("I feel 60% → log 80%"), NOT on nudging myself to introspect better (introspection is unreliable per [[calibration-research-verdicts]]). Elicitation quality and logging compliance dominate any new metric — inputs > downstream math. Everything stays pure `std`, no new crate deps, no LLM/network in `scoring.rs` (elicitation lives in the `predict` workflow layer).

**Tier 1 — DONE (2026-06-07):**
- `scoring::calibration_eprocess` (mixture of betting martingales over a fixed λ-grid, |λ|<1; e→p via 1/e) + `eprocess_pvalue`. Report shows "Is it real? e-value …". On my real n=8 ledger it correctly says **e=1.2, no evidence yet** — the −18pt gap is NOT real at this n.
- `scoring::fit_recalibration` (ridge-shrunk Newton logistic `σ(a+b·logit p)`) + `Recalibration::apply`. Surfaced in report + MCP `recalibrate` tool. **Gated on evidence** (e≥`RECAL_MIN_E`=3 AND n≥`RECAL_MIN_N`=6) so it never "corrects" on noise — found+fixed a bug where it printed a correction table under "no evidence". Mirrored into the Python binding. 37 Rust + 19 py tests green, clippy/fmt clean.

**Tier 2 — FULLY DONE (2026-06-07).** Two passes:
- *Small-sample/over-time trio:* `brier_ci_bootstrap` (seeded SplitMix64 percentile band — reproducible), `ewma_brier` (recency Brier, ~5-prediction half-life, *descriptive* "Lately … → improving/steady/slipping", NOT a CUSUM alarm — control charts false-alarm at an agent's n), `distinct_forecasts` (confidence-vocabulary).
- *The three deferred items, all built:* (1) **selective-prediction** `risk_coverage` (risk–coverage curve + AURCC; report shows "act on all X% · surest half Y% → …"); (2) **stakes-weighting** — a `stake` field on `Claim` (serde-default 1.0, `skip_serializing_if` so old ledgers stay byte-identical; legacy-load test guards it) + `brier_weighted`, with `--stake`/MCP `stake` wiring and a "Stake-weighted vs flat" report line shown only when stakes vary; (3) **elicitation** — `dialectical_mean` (Herzog–Hertwig crowd-within) + `compose_reasoning` (weaves outside-view reference class + the two estimates into `because`), wired into `add --second-prob/--reference-class` and MCP `predict`, with the protocol taught in the `/predict` command + `calibration-protocol` skill. All surfaced + mirrored to the Python binding. **45 Rust + 25 py + 5 e2e tests green, clippy/fmt clean.**

**Tier 3 — FULLY DONE (2026-06-08).** The win was *disciplined reuse*, not new math — exactly one new pure fn shipped. (1) **Conformal interval recalibration** `conformal_width_factor` (the numeric analogue of the binary recalibration map: multiply your half-widths by the split-conformal quantile of standardized residuals `|value−center|/half_width` to hit nominal coverage; report shows "Recalibration: WIDEN/SHARPEN — multiply half-widths by ×N", gated on real evidence). (2) **Small-sample correction via reuse:** the binary `calibration_eprocess` run on interval coverage (`prob=level`, `outcome=contained`) is the anytime-valid gate; `shrink_toward(k,n,0.5,1)` = Jeffreys mean `(k+½)/(n+1)` de-noises the coverage point. (Researched, then *avoided* a risky incomplete-beta impl — Wilson≈Jeffreys at small n, and the e-process is the better, peek-proof gate.) (3) **Per-kind multicalibration** = `calibration_eprocess` per `kind:` → verdict names the worst *really*-miscalibrated kind; anytime-valid ⇒ tiny subgroups can't false-alarm (the exact small-group failure the literature warns about). (4) **Cox slope/intercept** = `fit_recalibration`'s `(a,b)`, just named+read aloud (slope b<1 ⇒ too extreme; intercept ⇒ over/under-stating). **CRPS: DECLINED with cause** — for interval-format data the Winkler score already *is* the CRPS specialization (WIS→CRPS as #intervals→∞); CRPS would need a distribution shape I never logged = more math, less truth. **46 lib + 6 e2e + 26 py tests green, clippy/fmt clean.** Seed gained 4 too-tight intervals so the demo's coverage e-process crosses the bar and shows the ×2.76 WIDEN correction.

**No tiers remain.** The build is at saturation per the original exhaustion check; further work is polish or new research, not backlog.

**Post-Tier-3 (2026-06-08): README polish done; then a fresh research pass found the frontier has MOVED off scoring → decision-coupling, and I BUILT the answer.** The **decision gate** `scoring::decide` (recalibrate the stated p, then Chow's reject threshold `τ = 1 − verify_cost/stake` → Proceed/Verify/Abstain) closes the loop the 2026 literature says agents skip (verbalize uncertainty, act anyway). CLI `ana decide`, MCP `decide` tool, Python `ana.decide`; shared evidence gate `report::earned_recalibration`. See [[calibration-research-verdicts]] (Phase-2 + BUILT entry). This was the genuinely-new direction; the scoring math itself stays saturated.

**Resolution-discipline diagnostic (2026-06-08) — the one honest gap left, now closed.** The tool scored only the *resolved* subset and said nothing about whether that subset was a fair sample → it could flatter me (the file-drawer / non-random-censoring failure; live research, e.g. arXiv 2512.07575, 2312.13720). Built: `scoring::mean_boldness` (outcome-free `mean(max(p,1−p))`, so it works on still-OPEN claims) + `scoring::asmd` (absolute standardized mean difference — the textbook covariate-balance / MNAR effect size, flagged at >0.1, an effect size not a test so no small-n false alarm). `report::ResolutionDiscipline` surfaces resolution rate, overdue count (needed threading `today: NaiveDate` through compute/render/render_json + call sites), and the boldness ASMD of graded-vs-ungraded calls, rendered up top as the honesty caveat. Also added a 4th Monte-Carlo study proving the decision gate lowers expected cost (100% win-rate, ~8% cheaper). **51 lib + 8 e2e + 28 py tests; validate_guarantees 4/4.** The most on-thesis addition possible — it makes the mirror *harder to fool*, which is the whole point.

Current real self-data: ~8 resolved, **underconfident ≈ −18pts** (right ~81% vs ~64% claimed) — but n far too small to act on yet; the e-process/bootstrap CI are exactly what tell me whether that gap is real. See [[calibration-research-verdicts]].
